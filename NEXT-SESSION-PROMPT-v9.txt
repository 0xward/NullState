   tree already ships a complete `bow/` animation folder —

     lookup was `animKey === 'idle'||'walk' ? 'walkcycle' : animKey`
     — added a `animKey==='shoot' ? 'bow'` branch for the same reason.

   generically resolve `animKey = wDef.anim` from whichever weapon is
   equipped and look up `LPC_HERO[animKey]`, so `shoot` slotted in
   without touching that logic. marketplace-items.js already tagged
   `hunters_bow` with `behavior:'ranged'` for the separate arrow/FX
   system in game.js — untouched, still fires arrows same as before,
   this only adds the visual draw-pose+bow overlay under it.
   **Verified via offline PIL composite** (frame extraction + alpha
   stacking in the exact same order as drawLPCComposite: body, then
   legs/torso/feet/head armor, then weapon overlay) — down-facing,
   mid-draw frame, leather armor: renders as an archer holding a
   drawn bow, armor layers align correctly, no obvious mis-registration.
   **NOT YET verified inside the real canvas/game loop** (no
   npm/browser access this session) — this is the top item for next
   session's verification ladder, see below.
