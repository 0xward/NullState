## The "outdoor preview" requirement — read carefully, this is new scope this session

The user explicitly wants the pre-dungeon preview (in `DungeonGame.tsx`'s title screen, rendered by
`drawPreview()`/`paintPreview()` in `game.js`) to **not go stale/disconnected** once gear visuals
exist. Concretely:

- `drawPreview(elId, cfg)` currently does: grab the idle sheet's first frame → auto-crop to the
  sprite's opaque bounding box → scale into an 88x88 box → stamp it into a small canvas → done. It's
  a static single-frame snapshot, not animated, and has ZERO awareness of equipped gear (makes sense,
  since gear currently has no visual representation at all — see below).
- Once the weapon/armor layering system (next section) exists, **this preview needs to call the same
  layering/compositing function** used in the dungeon's `Player.draw()`, not a separate reimplementation. Concretely: refactor whatever composes body+armor+weapon in `entities.js` into a
  function that takes a canvas context + position + scale and just draws the composed result — call
  that same function from both `Player.draw()` (dungeon, animated) and `drawPreview()` (outdoor,
  static single frame). This keeps them in sync automatically instead of two divergent code paths that
  drift out of sync over time (which is exactly the "kaku"/disconnected feeling the user is trying to
  avoid).
- Since there's only 1 hero now, `paintPreview()` collapses to a single call, but it should still show
  **whatever the player currently has equipped** (mainhand + body slot from `G.equipment.equipped`),
  not just a bare body. Read `eq.mainhand`/`eq.body` the same way `applyEquipment()` does.
- This preview needs to read equipment BEFORE `G` (the live game state) necessarily exists yet (it's
  the title screen, pre-descend) — check how `G.equipment`/owned gear is currently loaded/persisted at
  boot (likely Firebase load happens before title screen paints, but verify, don't assume) so the
  preview shows accurate gear and doesn't flash bare-body-then-pop-in-gear.
